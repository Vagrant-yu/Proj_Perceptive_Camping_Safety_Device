### 2026 嵌入式大赛 - 野外生存安全系统 - 需求功能矩阵

#### 第一维度：全域感知与防御 (Perception & Defense)

*解决“未知恐惧”与“物理入侵”*


| 用户痛点 (Pain Points) | 场景描述 (Scenario) | 技术需求转化 (Technical Requirements) |
| :--- | :--- | :--- |
| **黑暗中的未知恐惧** | 深夜听到帐篷外有异响（枯枝断裂、沉重脚步），无法判断是风吹、小动物还是猛兽/坏人，不敢入睡或频繁惊醒。 | **Edge AI 听觉分类 (TinyML)：**<br>1. **训练目标**：识别脚步声、低吼声、拉链声 vs 风雨声、流水声。<br>2. **性能指标**：本地推理延迟 < 500ms，准确率 > 90%。<br>3. **传感器**：I2S 数字麦克风 (如 INMP441)。 |
| **防御边界缺失** | 昂贵装备（自行车、摄影包）必须放在帐篷外前厅，或挂在树上，担心被偷或被野兽翻找食物。 | **分布式震动“结界” (Perimeter Defense)：**<br>1. **无线感知**：开发纽扣电池供电的 BLE/2.4G 震动标签。<br>2. **联动逻辑**：标签检测到持续震动 -> 触发主机报警 + 指向性强光威慑。 |
| **结构崩塌风险** | 遭遇强风或积雪，无法预知帐篷骨架的承受极限。担心睡梦中帐篷杆断裂刺伤人体或直接倒塌。 | **结构健康监测 (SHM)：**<br>1. **姿态解算**：利用 IMU (MPU6050) 监测帐篷主杆的弯曲角度与震动频率。<br>2. **AI 异常检测**：区分“正常风吹晃动”与“塑性形变前兆”，提前预警撤离。 |
| **隐形雷击威胁** | 在高海拔或空旷营地，雷暴云层逼近时往往无声无息，等到听到雷声或看到闪电时已来不及寻找掩体。 | **电磁风暴预警 (Lightning Detection)：**<br>1. **电磁感知**：集成 AS3935 雷电传感器，检测 40km 内雷电电磁脉冲。<br>2. **距离估算**：计算雷暴锋面的逼近速度，提供逃生时间窗口。 |

#### 第二维度：离网通信与定位 (Communication & Location)

*解决“失联无助”与“迷失方向”*

| 用户痛点 (Pain Points) | 场景描述 (Scenario) | 技术需求转化 (Technical Requirements) |
| :--- | :--- | :--- |
| **通信孤岛 (失联)** | 深山无 4G/5G 信号。一旦受伤、生病或遇到泥石流，手机变砖头，无法向外界求救。 | **LoRa Mesh 组网 (Off-Grid Network)：**<br>1. **长距通信**：使用 LoRa (SX1262) 实现 3-5km 点对点通信。<br>2. **自动信标**：触发 SOS 时，自动广播 GPS/北斗坐标及遇险类型编码。<br>3. **文字通联**：支持通过蓝牙连接手机 APP，发送离网短消息给队友。 |
| **夜间近场迷路** | 夜间起夜或取水，离开帐篷仅几十米，因无参照物且光线漆黑，回头找不到帐篷（“鬼打墙”）。 | **归巢信标 (Homeward Beacon)：**<br>1. **RSSI 测距**：根据用户手环/手机蓝牙信号强度判断距离。<br>2. **灯塔效应**：用户离开 >10米 且回头面向帐篷时，帐篷开启高亮频闪指引方向。 |
| **无声求救** | 遇到歹徒或极度虚弱无法呼喊时，需要一种被动的、外界可感知的求救方式。 | **SOS 光语系统 (Visual SOS)：**<br>1. **被动触发**：AI 识别到特定口令（如连续轻叩麦克风）或跌倒。<br>2. **光语编码**：控制高亮 LED 发送标准摩尔斯电码（三短三长三短）。 |

#### 第三维度：生存环境保障 (Environmental Support)

*解决“隐形杀手”与“能源焦虑”*

| 用户痛点 (Pain Points) | 场景描述 (Scenario) | 技术需求转化 (Technical Requirements) |
| :--- | :--- | :--- |
| **隐形杀手 (CO/失温)** | 寒冷天气紧闭帐篷导致通风不良，引发一氧化碳中毒（无色无味）；或内部结露打湿睡袋导致失温。 | **生存环境监测 (Critical Environment)：**<br>1. **多维传感**：集成 CO 传感器 + 温湿度传感器。<br>2. **露点算法**：计算露点温度，预测结露风险。<br>3. **主动干预**：危险阈值触发换气风扇或声光唤醒。 |
| **电量焦虑** | 户外充电宝资源宝贵。传统设备耗电快，关键时刻（如求救时）没电是致命的。 | **微电子电源管理 (Smart PMIC)：**<br>1. **能量采集**：支持便携式太阳能板 MPPT (最大功率点追踪) 充电。<br>2. **动态休眠策略**：电量 <20% 时，切断舒适性功能（风扇/灯光），仅保留 AI 监听与 LoRa 信标，确保“活着”。 |

### 💡 核心功能优先级建议 (MVP Roadmap)

在这个庞大的列表中，为了应对竞赛有限的开发时间，建议按以下优先级进行开发：

1. **P0 (基石 - 必须完成):**
* Edge AI 听觉分类 (核心赛题点)
* LoRa 离网通信 (核心功能点)
* CO/温湿度监测 (基础生存点)


2. **P1 (亮点 - 强力加分):**
* 结构健康监测 (IMU) **或** 雷电预警 (二选一，体现差异化)
* 智能电源管理 (MPPT) (体现微电子专业能力)


3. **P2 (锦上添花 - 有空再做):**
* 分布式震动标签 (涉及额外的硬件节点开发，工作量大)
* 归巢信标 (逻辑简单，最后加上即可)
